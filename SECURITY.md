# 安全问题

## 支持版本

当前维护 2.0.x 的最新版本。旧补丁版本、Snapshot 和 RC 不长期维护。

## 怎么报告漏洞

请使用 GitHub 仓库里的 Security / Report a vulnerability 私下报告，不要直接开公开 Issue。

报告里最好写清：

- 受影响版本和模块；
- 怎么复现；
- 实际影响；
- 有没有已经公开或被利用；
- 你想到的临时处理办法。

不要在报告、日志或示例仓库里放真实密码、Token、私钥、证书和配置内容。

修复准备好以前，我们不会要求报告者公开漏洞。是否申请 CVE、什么时候公开，确认影响后再沟通。

## 当前安全边界

- 配置内容按明文业务数据处理，需要用 TLS、数据库权限和主机权限保护。
- Token、密码和私钥不能写进 URL、日志和指标。
- 多 Gateway 切换仍然只能保留一个活动 Session。
- PostgreSQL 不在 2.0 的官方支持范围内。

更完整的说明见[架构设计](doc/architecture.md)和[功能说明](doc/features.md)。
